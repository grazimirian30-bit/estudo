Simulado – Engenharia de Software (USF)

30 questões • Múltipla escolha • Nível graduação

1) Uma empresa iniciou o desenvolvimento de seus sistemas sem padrões definidos. Com o crescimento, surgiram falhas frequentes, retrabalho e dificuldade de manutenção. Nesse cenário, a Engenharia de Software torna-se essencial porque:

Gabarito: C
A Engenharia de Software existe para controlar complexidade, evolução e qualidade por meio de processos e métodos.

2) Tecnologias como computação em nuvem, Big Data e IoT tornaram-se estratégicas para as organizações principalmente porque:

Gabarito: B
Essas tecnologias permitem inovação, crescimento rápido e novos serviços digitais.

3) Uma empresa automatizou seus processos internos, mas manteve a mesma cultura e modelo de negócio. Os ganhos competitivos foram limitados. Isso indica que a transformação digital:

Gabarito: C
Transformação digital envolve pessoas, cultura, estratégia e tecnologia.

4) Em um mercado altamente digitalizado, empresas competitivas destacam-se principalmente pela:

Gabarito: B
Na economia digital, o foco está na experiência do cliente.

5) O alinhamento entre TI e negócio é fundamental porque:

Gabarito: C
A TI deixou de ser suporte e passou a ser elemento estratégico.

6) Ao avaliar um sistema de informação, percebe-se que seu funcionamento depende tanto da tecnologia quanto das pessoas e processos. Isso reforça que um sistema de informação é composto por:

Gabarito: B
Sistemas de informação são sociotécnicos.

7) Durante a fase inicial de um projeto, a equipe busca entender o que o sistema deve fazer e quais problemas resolver. Essa atividade caracteriza a fase de:

Gabarito: B
A análise define requisitos e funcionalidades.

8) A fase em que o sistema é efetivamente construído com base nos requisitos definidos corresponde ao:

Gabarito: B
Desenvolvimento é a implementação do sistema.

9) Quando dados brutos são organizados e interpretados para apoiar decisões, eles passam a ser considerados:

Gabarito: B
Informação é dado com significado.

10) Um gestor utiliza relatórios para identificar tendências e apoiar decisões futuras. Nesse contexto, os sistemas de informação têm como principal função:

Gabarito: B
Sistemas apoiam, não substituem, decisões humanas.

11) Sistemas que registram vendas, pagamentos e movimentações rotineiras atendem principalmente ao nível:

Gabarito: B
SPT lidam com operações do dia a dia.

12) Decisões que afetam o futuro da organização e possuem impacto de longo prazo são classificadas como:

Gabarito: A
Decisões estratégicas definem o rumo da organização.

13) Sistemas de Informação Gerencial utilizam dados principalmente:

Gabarito: C
SIG consolidam dados dos SPT.

14) Sistemas de Apoio à Decisão são mais indicados quando:

Gabarito: C
SAD apoiam decisões não estruturadas.

15) Sistemas de Apoio Executivo são voltados principalmente para:

Gabarito: C
SAE atendem o nível estratégico.

16) O conceito de social business está relacionado:

Gabarito: B
Social business integra empresa e sociedade.

17) Segundo Pressman, um software é composto por:

Gabarito: D
Documentação faz parte do produto de software.

18) A Engenharia de Software adota uma abordagem sistemática para:

Gabarito: B
Processos bem definidos aumentam a qualidade.

19) A reutilização de componentes de software contribui principalmente para:

Gabarito: C
Reutilização aumenta produtividade.

20) A Engenharia de Software é considerada multidisciplinar porque envolve:

Gabarito: B
A ES integra áreas técnicas e organizacionais.

21) Um dos principais desafios atuais da Engenharia de Software é:

Gabarito: B
A demanda cresce mais rápido que a produção.

22) A manutenção de software tende a ser cara porque:

Gabarito: C
Necessidades mudam ao longo do tempo.

23) A organização da Engenharia de Software em camadas tem como objetivo:

Gabarito: B
As camadas garantem controle e qualidade.

24) Um processo de software mal definido impacta diretamente:

Gabarito: A
Processo ruim gera software ruim.

25) As atividades genéricas do processo de software incluem:

Gabarito: D
São as etapas básicas do processo.

26) Um sistema atende todas as especificações, mas não resolve o problema do usuário. Isso indica falha na:

Gabarito: B
Validação verifica se o software resolve o problema certo.

27) Um software tecnicamente correto, mas difícil de usar, falha principalmente no critério de:

Gabarito: B
Usabilidade é essencial para aceitação.

28) A documentação de software é importante porque:

Gabarito: C
Documentação reduz dependência de pessoas.

29) A evolução do software ocorre principalmente porque:

Gabarito: D
Contextos e necessidades evoluem.

30) Um software pode ser considerado bem-sucedido quando:

Gabarito: A
Aceitação do usuário é o principal critério.
